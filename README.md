# TestForZMedia — Tiny Army Battle Simulator

Tiny Army Battle Simulator — это мобильный симулятор тактического боя. Проект представляет собой реализацию тестового задания с акцентом на **чистую архитектуру**, **реактивное программирование** и производительность на мобильных платформах.

## 🏗 Архитектурный стек (Advanced Patterns)

Проект построен на принципах **Separation of Concerns** и слабой связанности систем:

*   **Dependency Injection ([Zenject](https://github.com)):** Централизованная сборка через `CoreInstaller.cs`. Использование `ProjectContext` и `SceneContext` исключает использование Singletons и `FindObjectOfType`.
*   **Interface-Driven Interaction:** Системы (здоровье, движение, атака) реализованы через интерфейсы (`IHealth`, `IMove`, `IAttack`). Это упрощает Unit-тестирование и внедрение Mock-объектов.
*   **Reactive State Management ([UniRx](https://github.com)):** Обновление UI и логика работают по событийной модели. Подписки на изменения состояния (HP, GameState) в `UnitHealth.cs` и `UiPresenter.cs` минимизируют нагрузку на CPU.
*   **Value-Type Data Contract:** Данные юнитов хранятся в `struct UnitData`, что исключает аллокации в куче (GC) во время боя.

## ⚡ Оптимизация и ресурсы

Проект ориентирован на **60 FPS** на мобильных устройствах:

1.  **MaterialPropertyBlock:** Изменение цвета юнитов без потери GPU-батчинга + custom shader для URP использую `#pragma multi_compile_instancing`. Все 40 юнитов рендерятся в 2 Draw Call.
2.  **Tuple-Key Dictionary:** Поиск целей в `BattleManager.cs` через `(ArmyType, Id)` обеспечивает **O(1)** доступ.
3.  **SqrMagnitude Optimization:** Расчёты дистанций используют квадрат расстояния, избегая дорогостоящей операции `Mathf.Sqrt`.
4.  **CompositeDisposable:** Автоматическая очистка всех UniRx-подписок для предотвращения утечек памяти.

## ✨ Дополнительная функция: Система формаций

В проект интегрирована система тактических построений:

*   **Data-Driven Formations:** Конфигурации хранятся в `FormationConfigSO.cs`. Новые паттерны создаются без изменения кода.
*   **Dynamic Repositioning:** Метод `ChangeFormation()` перестраивает армию в меню предпросмотра без респавна объектов.
*   **Collision Avoidance:** Оффсеты кратны 2 единицам, что предотвращает "слипание" коллайдеров при спавне.

## 🚀 Как запустить

1.  Откройте проект в **Unity 6000.2f** или новее.
2.  Убедитесь, что все конфиги `ScriptableObject` назначены в инспекторе `CoreInstaller`.
3.  Откройте сцену `BattleScene` и нажмите **Play**.
4.  В меню измените состав армий, выберите формации и нажмите **Start Battle**.

---

## ➕ Как добавить новых юнитов и формации

1.  Создайте новый вариант префаба Unit в папке Prefabs.
2.  Назначьте на префаб новый mesh.
3.  Создайте новые конфиги Create->Configs->UnitConfigs.
4.  Настройте модификаторы и поля в конфиге.
5.  Добавьте в `CoreInstaller` в инспекторе в соответствующие массивы.
6.  В формации добавьте оффсеты Vector2 (y для z).
7.  Наслаждайтесь.
